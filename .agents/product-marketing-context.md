# Product Marketing Context

*Last updated: March 2026*

## Product Overview
**One-liner:** The fastest way for personal trainers to build client programs — type how you think, get polished programs instantly.

**What it does:** Wagmi Fit is a workout programming tool for independent personal trainers. Trainers type in natural shorthand — messy, fast, the way they actually think — and the system instantly structures and formats it into polished, client-ready programs. No templates, no drag-and-drop builders, no reformatting. Just type and it's done.

**Product category:** Personal trainer software / workout programming software / coaching software

**Product type:** B2B SaaS

**Business model:** Subscription (free during beta; paid tiers TBD). Domain: wagmi.fit

---

## Target Audience
**Target companies:** Independent personal trainers — solo coaching businesses, not gym chains.

**Decision-makers:** The trainer themselves (solo operator). No procurement process. Direct purchase.

**ICP:** Independent personal trainer, 1–50 clients, online or hybrid coaching, age 25–38, internet-native, uses modern tools, values speed over feature bloat. Manages their own business admin. Likely using a mix of Trainerize/TrueCoach + Google Sheets + ChatGPT copy-paste to get by.

**Primary use case:** Building and delivering workout programs to clients faster — removing the administrative overhead of formatting, structuring, and organizing programs.

**Jobs to be done:**
- Program more clients without adding more admin hours
- Deliver polished, professional client experiences without manual formatting
- Spend time coaching, not clicking through software

**Use cases:**
- Writing a weekly block for 10-20 clients with quick natural-language input
- Delivering structured programs to clients on their phone
- Managing client context (injuries, PRs, preferences) that informs programming
- Iterating mid-week when client needs change

## Problems & Pain Points
**Core problem:** Trainers know exactly what to program. Existing software makes the act of programming slow, rigid, and full of friction. The bottleneck isn't coaching knowledge — it's the interface.

**Why alternatives fall short:**
- **Trainerize / TrueCoach / Everfit:** Click-heavy template builders designed for gym enterprise, not solo coaches. Drag-and-drop feels powerful until you've done it 200 times. Built for 2013, not 2025.
- **ChatGPT + Spreadsheets:** Disconnected workflow. Copy-pasting between tools. Nothing is integrated from brain to client delivery.
- **Excel / Google Sheets:** Requires constant formatting, no client delivery layer, breaks at scale.

**What it costs them:**
- 45 minutes to 2 hours per week per client on programming and delivery
- Across 15 clients: up to 30 hours/month spent formatting, not coaching
- Mental energy wasted on interfaces instead of training science
- Growth ceiling — can't take on more clients without more admin hours

**Emotional tension:** "I love coaching. I hate that my software makes me feel like a data entry clerk."

## Differentiation
**Key differentiators:**
- Type in natural shorthand — the system parses it, not the trainer
- Zero reformatting — structured output is automatic, not manual
- Speed as a first principle — built around the trainer's time, not feature count
- Not a chatbot / prompt interface — trainers don't "describe" to AI, they just type their program

**How we do it differently:** Accepts the messy way trainers already think and write. Treats the trainer's brain as the input, not a template library.

**Why that's better:** Faster programming → iterate more → better programs → more clients → sustainable business.

**Why customers choose us:** The only tool where typing rough notes results in polished client programs with zero intermediate steps.

## Objections

| Objection | Response |
|-----------|----------|
| "My clients are already set up in Trainerize / I'd have to migrate" | Migration is a one-time cost. The ongoing time savings compound every single week. |
| "Is it really faster or just different?" | The core workflow is: type shorthand → done. No template browsing, no drag-and-drop, no reformatting. The speed difference is immediate. |
| "What if it structures something wrong?" | You review the output before delivery. It's a speed layer, not an autonomous system — you stay in control of the program. |
| "I've tried other tools and gone back to spreadsheets" | Wagmi Fit isn't a feature-bloated platform. It's a single workflow: type → structured → delivered. If you can type, you can use it. |

**Anti-persona:**
- Gym chains or franchise operations with admin staff (we're built for solo operators)
- Trainers with 1-3 clients who don't feel the admin time pressure
- Trainers who want heavy template libraries and prefer visual drag-and-drop builders
- Enterprise fitness software buyers looking for HR integrations, payroll, class scheduling

---

## Switching Dynamics
**Push (away from current tools):** Legacy platforms are slow; spreadsheet workflows break at scale; ChatGPT copy-paste is a workaround, not a solution; trainers feel like they're doing data entry.

**Pull (toward Wagmi Fit):** Programming at the speed of thought; no reformatting; polished client output; designed for how trainers actually work, not how gym admins work.

**Habit (keeps them stuck):** Existing client setups in current tools; clients familiar with current delivery format; workflows already built around workarounds; sunk cost in learning current platform.

**Anxiety (about switching):** Will clients tolerate a new app? Will it actually be faster after onboarding? What if the AI structures something incorrectly? Is the company going to be around?

**Words to avoid:**

- Revolutionary, game-changing, disrupt
- Synergy, leverage, enterprise
- Chatbot, AI assistant, AI interface
- "Enter a prompt"

**Glossary:**
| Term | Meaning |
|------|---------|
| Programming | Writing workout plans for clients (not software programming) |
| Block | A training phase (e.g., hypertrophy block, strength block) |
| AMRAP | As Many Reps As Possible — common trainer shorthand |
| RPE | Rate of Perceived Exertion — effort scale |
| Client-ready | Formatted, structured, deliverable to a client without further editing |
